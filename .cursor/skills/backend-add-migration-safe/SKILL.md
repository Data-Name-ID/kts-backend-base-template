# Backend Add Migration Safe

Проводить изменения схемы так, чтобы миграции были обратимыми, предсказуемыми и безопасными для выката.

## Workflow

1. Зафиксировать изменение схемы.
- Описать, что меняется: таблицы, колонки, индексы, ограничения, enum, данные.
- Проверить влияние на API, repo/accessor и manager/service слой.

2. Обновить модель и контракты слоя данных.
- Сначала обновить модельную часть в проектном паттерне.
- Проверить, что naming и связи согласованы с существующей схемой.

3. Создать миграцию.
- `uv run alembic revision --autogenerate -m "<краткое описание>"` из директории `backend/`.
- Файл создаётся в `app/core/models/migrations/versions/`.
- Не редактировать старые миграции без явной необходимости.

4. Доработать миграцию вручную.
- Проверить `upgrade()` и `downgrade()` на полноту.
- Явно обработать enum-изменения и data-migration сценарии.
- Сохранить атомарность и предсказуемый порядок шагов.

5. Проверить безопасность применения.
- Применить: `uv run alembic upgrade head`.
- Откатить: `uv run alembic downgrade -1`; убедиться, что состояние схемы восстановлено.
- Проверить совместимость кода с новой схемой после upgrade и после downgrade.
- Убедиться, что naming_convention из `BaseModel` соблюдено (индексы: `ix_`, уникальные: `uq_`, FK: `fk_`, PK: `pk_`, CHECK: `ck_`).

6. Проверить эксплуатационные риски.
- Оценить блокировки, длительные операции, влияние на большие таблицы.
- Для тяжелых изменений предусмотреть staged rollout (expand/migrate/contract), если это требуется.

7. Закрыть задачу тестами и проверками.
- Обновить тесты слоя данных и API при необходимости.
- `uv run ruff check .` + `uv run mypy .` + `uv run pytest`.

## Anti-Patterns

- Полагаться только на автогенерацию без ревью миграции.
- Оставлять пустой или нерабочий `downgrade()`.
- Смешивать несвязанные изменения в одной миграции.
- Ломать обратную совместимость без плана выката.

## Definition Of Done

- Миграция корректно применяется и откатывается.
- Изменения схемы совместимы с кодом приложения.
- Риски выката оценены и задокументированы.
- Локальные проверки и тесты проходят.
